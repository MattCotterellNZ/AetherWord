What is Aetherword?

Aetherword is a password manager, like keepass or LastPass. How Aetherword differs from the others is in where your passwords are stored: in the aether (i.e. nowhere).

Aetherword instead generates your passwords for you to use, based off a secret "master password" that only you know and the domain name of the website you want to log in to. The website-specific password is then created using industry standard cryptography and encoding, ready for you to copy and paste into your favorite websites. This is done in such a way as to make it basically impossible to reverse back to your secret "master password", thus keeping you safe on all other websites, even if your password is leaked by another. All this, while only having to remember one password, and all without a single password list to look after.

Cryptography

Aetherword uses industry standard cryptographic protocols such as PBKDF2 and HMAC-SHA256 to keep your master password safe. The idea behind it is actually remarkably simple.

First, a colour is generated based off your password using a single round of PBKDF2, discarding all bytes except the first. This is used to create the colour you see around your password as you type it, so you know straight away if you've mistyped it. The collision likelyhood of this is so high (only 256 possible colours) that it is completely impossible to reverse engineer your password simply by knowing the colour.

Next, the master password is stored temporarily in memory while we await which websites you wish to log in to. Once a website domain is entered, such as "facebook.com", the "master password" is hashed on a seperate "web worker" thread using 10,000 rounds of PBKDF2-HMAC-SHA256, using the domain as the salt. 128 bits of hash data is then extracted and encoded using a process very similar to ascii85 encoding (notably the "z" and "y" rules are ignored) and returned to the user as a 20-digit, random-looking password. This process is very computationally expensive, making brute-forcing the master password extremely difficult (usually in the realm of centuries to crack). Any competent web developer knows that it is essential to hash passwords when storing them in their database anyway (making cracking even more difficult), but this way you're protected on other sites even if they have screwed up.
